# Backend Instructions — Falcon Supporter

## Vai trò

Backend là trung tâm xử lý của hệ thống. Nhận request từ frontend, gọi AI Agent, quản lý Mock PR workflow, và đọc/ghi dữ liệu file-based (CSV + JSON).

## Tech Stack

- **Runtime**: Node.js (Express) hoặc Python (FastAPI) — chọn một.
- **Data**: File CSV (`players.csv`) + JSON (`knowledge_base.json`, `conversations.json`, `pending_reviews.json`).
- **AI**: Gọi LLM API bên ngoài (OpenAI / Claude / Gemini) qua HTTP.

## API Endpoints cần triển khai

### `POST /api/auth`
- **Input**: `{ player_id: "P001" }`
- **Logic**:
  1. Tra cứu `player_id` trong `players.csv`.
  2. Nếu tồn tại → set cookie `player_id`, trả về thông tin player + lịch sử chat từ `conversations.json`.
  3. Nếu không tồn tại → trả `404`.
- **Output**: `{ player: { id, username, gold, level, status }, history: [...messages] }`

### `POST /api/chat`
- **Input**: `{ player_id: "P001", message: "Tôi bị mất 500 gold" }`
- **Logic**:
  1. Đọc player data từ CSV.
  2. Gọi `search_kb(message)` — keyword matching trên `knowledge_base.json`.
  3. Gọi `analyze_message(message, player_data, kb_results)` — gọi LLM API.
  4. Lưu message + AI reply vào `conversations.json`.
  5. Nếu `confidence > 0.8` → tạo Mock PR trong `pending_reviews.json`.
  6. Trả kết quả về frontend.
- **Output (confidence > 0.8)**:
  ```json
  {
    "reply": "Tôi đã kiểm tra tài khoản và phát hiện...",
    "status": "pending_review",
    "review_id": "R001",
    "diff": {
      "player_id": "P001",
      "field": "gold",
      "before": 15000,
      "after": 15500,
      "reason": "Hoàn 500 gold do lỗi sau update"
    }
  }
  ```
- **Output (confidence ≤ 0.8)**:
  ```json
  {
    "reply": "Tôi chưa thể xử lý tình huống này, đang chuyển cho đội CSKH...",
    "status": "escalated"
  }
  ```

### `GET /api/pending-reviews`
- **Logic**: Đọc `pending_reviews.json`, trả về danh sách Mock PR chưa resolve.
- **Output**: `{ reviews: [{ review_id, player_id, field, before, after, reason, created_at }] }`

### `POST /api/resolve-pr`
- **Input**: `{ review_id: "R001", action: "approve" | "reject" }`
- **Logic**:
  - `approve` → cập nhật giá trị trong `players.csv`, đánh dấu PR là `merged` trong `pending_reviews.json`.
  - `reject` → đánh dấu PR là `rejected`, không thay đổi CSV.
- **Output**: `{ status: "merged" | "rejected" }`

## AI Agent Functions

### `search_kb(message)`
- Tách từ khóa từ message.
- So sánh với mảng `keywords` trong mỗi entry KB.
- Trả về top matches (có thể dùng simple overlap count).

### `analyze_message(message, player_data, kb_results)`
- Gọi LLM API với prompt gồm: system role + player context + KB matches.
- Yêu cầu LLM trả JSON: `{ reply, confidence, db_changes }`.
- Parse kết quả, validate confidence nằm trong [0, 1].

## Quy tắc quan trọng

- **KHÔNG BAO GIỜ** tự ý ghi vào `players.csv` từ logic chat. Mọi thay đổi phải qua Mock PR.
- Luôn enable **CORS** để frontend ở port khác có thể gọi API.
- Lưu conversation mỗi khi có message mới (cả user lẫn AI reply).
- Đọc `player_id` từ cookie hoặc body request.
