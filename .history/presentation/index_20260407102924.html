✅ Luồng Auto-Resolve

Khi AI tìm thấy tình huống tương tự trong Knowledge Base

  1. User gửi tin nhắn mô tả vấn đề
  2. AI Agent tra cứu Knowledge Base → match!
  3. AI tạo Mock PR với db_changes
  4. Đội CSKH review trên giao diện Before/After Diff
  5. CSKH ấn Approve → DB cập nhật → Phản hồi user
Điều kiện: $$\text{confidence} > 0.8 \implies \text{Auto-resolve (tạo Mock PR)}$$

📨 Luồng Manual (Escalate)

Khi AI không tìm thấy case tương tự hoặc confidence thấp

  1. User gửi tin nhắn → AI không match Knowledge Base
  2. Chuyển tiếp sang hộp thư đội CSKH
  3. Đội CSKH xử lý thủ công & cập nhật DB
  4. Phản hồi kết quả cho user
  5. Feedback loop → AI học case mới → Cập nhật KB
Điều kiện: $$\text{confidence} \leq 0.8 \implies \text{Escalate (chuyển CSKH)}$$

🧮 Mô hình AI Confidence

Confidence score được tính dựa trên độ tương đồng:

$$\text{confidence}(q, c) = \frac{|K_q \cap K_c|}{|K_q \cup K_c|}$$

$K_q$: keywords trong câu hỏi user  •  $K_c$: keywords trong case KB

$\text{conf} > 0.8$
→ Tự động tạo Mock PR
→ CSKH chỉ cần review
$\text{conf} \leq 0.8$
→ Escalate đội CSKH
→ Xử lý thủ công
$$P(\text{auto-resolve}) = \frac{\sum_{i=1}^{N} \mathbb{1}[\text{conf}_i > 0.8]}{N} \xrightarrow{t \to \infty} 1$$

Tỉ lệ auto-resolve tăng dần khi Knowledge Base phát triển

🔄 Vòng lặp Feedback & Học tập

$$|KB_{t+1}| = |KB_t| + \Delta_{\text{new cases}}$$

Knowledge Base tăng trưởng sau mỗi case được resolve

  • Mỗi case resolve → Dữ liệu được đẩy cho AI Agent
  • AI Agent cập nhật Knowledge Base
  • Case mới trở thành tình huống mẫu tham chiếu
  • Lần sau gặp tình huống tương tự → AI tự xử lý
$$\lim_{t \to \infty} \frac{\text{auto-resolved}}{\text{total tickets}} \to 1$$

Hệ thống ngày càng thông minh, giảm dần sự can thiệp thủ công
