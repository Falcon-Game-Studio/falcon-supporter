# Frontend Instructions — Falcon Supporter

## Vai trò

Frontend là giao diện duy nhất mà người chơi (player) và đội CSKH tương tác với hệ thống. Bao gồm 2 phần chính: **Chat UI** và **Review UI** (giao diện duyệt Mock PR).

## Tech Stack

- **Framework**: React (hoặc HTML/JS đơn giản).
- **Giao tiếp**: REST API với backend (fetch / axios). Không dùng WebSocket.
- **State**: React state đơn giản (useState/useReducer). Không cần Redux.

## Màn hình chính

### 1. Màn hình xác thực — Hỏi Player ID

Khi user mở app lần đầu (chưa có cookie `player_id`):

- Hiển thị ô input "Nhập Player ID" + nút "Xác nhận".
- Gọi `POST /api/auth { player_id }`.
- Nếu thành công → lưu cookie `player_id`, chuyển sang Chat UI, hiển thị lại lịch sử chat cũ.
- Nếu thất bại (404) → hiển thị "Player ID không tồn tại".
- Nếu đã có cookie → bỏ qua bước này, gọi `/api/auth` để load lại history.

### 2. Chat UI

- **Danh sách tin nhắn**: Bubble user (bên phải, màu xanh), bubble AI (bên trái, màu xám).
- **Ô nhập**: Input text + nút Gửi.
- **Loading**: Khi chờ AI response, hiển thị spinner hoặc "AI đang xử lý...".
- **Kết quả**:
  - Nếu `status: "pending_review"` → hiển thị thêm badge "🟡 Đang chờ duyệt" + thông tin diff tóm tắt.
  - Nếu `status: "escalated"` → hiển thị badge "🔴 Đã chuyển CSKH".
  - Nếu `status: "merged"` → hiển thị badge "🟢 Đã xử lý xong".

### 3. Review UI (Giả lập PR) — Dành cho đội CSKH

- Gọi `GET /api/pending-reviews` để lấy danh sách Mock PR.
- Mỗi item hiển thị dạng **diff**:
  ```
  Player: P001 (DragonSlayer)
  Field:  gold
  ─────────────────────
  - Before: 15000       (màu đỏ / gạch ngang)
  + After:  15500       (màu xanh lá)
  ─────────────────────
  Reason: Hoàn 500 gold do lỗi sau update
  
  [✅ Approve]  [❌ Reject]
  ```
- Khi ấn Approve → `POST /api/resolve-pr { review_id, action: "approve" }`.
- Khi ấn Reject → `POST /api/resolve-pr { review_id, action: "reject" }`.
- Sau khi resolve → refresh lại danh sách.

## API Calls cần gọi

| Action | Method | Endpoint | Body |
|---|---|---|---|
| Xác thực | POST | `/api/auth` | `{ player_id }` |
| Gửi chat | POST | `/api/chat` | `{ player_id, message }` |
| Lấy pending PRs | GET | `/api/pending-reviews` | — |
| Approve/Reject | POST | `/api/resolve-pr` | `{ review_id, action }` |

## Quy tắc quan trọng

- **Cookie** `player_id` lưu ở browser, gửi kèm mọi request (hoặc đính trong body).
- Hiển thị lại **toàn bộ lịch sử chat** khi user quay lại (dữ liệu từ `/api/auth` response).
- Giao diện **không cần responsive** — chỉ cần hoạt động tốt trên desktop.
- Không cần animation phức tạp. Ưu tiên hoàn thiện chức năng trước.
